Database For A Mechanic Shop

Modeled and built a database for a Mechanic Shop using PostgreSQL. The database is used to track information related to customers, cars, service requests, car ownership, billing information. The project was completed in 3 phases:
1. Entity-Relationship (ER) Design: Created an ER-diagram using the requirement analysis given in the documentation
2. Schema Design: Created the relational schema using the ER-diagram from phase1
3. Implementation: Developed a client application using the Java Database Connector (jdbc) for psql. The client application supports specific queries and functionality relating to the Mechanic Shop.
